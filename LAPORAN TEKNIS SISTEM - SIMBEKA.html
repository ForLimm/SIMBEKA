Database & Data Integrity

💾 Skema Database & Driver

  • Driver Fleksibel: Sistem mendukung SQLite (default local development) dan MySQL / MariaDB (production).
  • Session & Cache: Sesi pengguna (`SESSION_DRIVER`) dan cache sistem (`CACHE_STORE`) dipusatkan pada database untuk stabilitas dan persistensi state login yang solid.
  • Queue Driver: Pengolahan antrean tugas di latar belakang (`QUEUE_CONNECTION`) dikelola via tabel database (`jobs`).

🛡️ Integritas Data & Relasi

  • SoftDeletes: Diimplementasikan pada data sensitif (`students`, `reports`, `archives`, `counseling_sessions`) untuk melindungi dari ketidaksengajaan penghapusan data.
  • Cascade Deletes: Pengaturan foreign key terkonfigurasi secara programmatis dan database-level cascade untuk membersihkan relasi terkait saat data induk dihapus.
  • Index Optimasi: Kolom foreign key penting (`student_id`, `teacher_id`, `academic_period_id`), status laporan, dan role user dilengkapi index untuk mempercepat loading data.

Security & Encryption

🔑 Enkripsi & Otorisasi

  • Enkripsi Recovery Code: Kode pemulihan siswa/guest di-enkripsi di database menggunakan algoritma AES-256-CBC via Crypt::encryptString untuk proteksi dari kebocoran data plain-text.
  • Hashing Satu Arah: Password user dan jawaban pertanyaan keamanan disimpan menggunakan hash algoritma bcrypt standar industri dengan cost factor 12.
  • Role-Based Access Control (RBAC): Hak akses super admin, guru bk, dan siswa diproteksi ketat menggunakan custom middleware RoleMiddleware dan GuestOnly.

🌐 HTTP Protection & Sanitasi

  • Security Headers Middleware: Otomatis menambahkan header keamanan di setiap response:
    • X-Frame-Options: SAMEORIGIN (Proteksi Clickjacking)
    • X-Content-Type-Options: nosniff (Proteksi MIME Sniffing)
    • X-XSS-Protection: 1; mode=block (Proteksi Cross-Site Scripting)
    • Referrer-Policy: strict-origin-when-cross-origin
    • Permissions-Policy: camera=(), microphone=(), geolocation=()
  • Sanitasi Input XSS: Setiap pengiriman form konsultasi atau laporan secara otomatis dibersihkan dari tag HTML menggunakan helper strip_tags().
  • Rate Limiting (Anti Brute Force): Pembatasan jumlah request percobaan login (5 kali/menit) dan pembuatan akun guest (10 kali/menit) untuk mencegah serangan brute force.

💬 Kebijakan Privasi Sesi: Self-Destruct Chat

Setelah kasus konsultasi online diselesaikan (status diubah menjadi resolved oleh Guru BK), sistem mengeksekusi penutupan sesi chat secara permanen. Seluruh pesan dalam sesi tersebut ditandai dengan flag is_destroyed = true pada database dan disembunyikan secara permanen dari query Eloquent menggunakan custom Scope visible(), menjamin privasi percakapan sensitif antara siswa dan konselor.

Glosarium Teknis (Bahasa Awam)

Untuk memudahkan komite sekolah, kepala sekolah, staf guru, dan pihak non-teknis memahami bagaimana sistem SIMBEKA dibangun, berikut adalah penjelasan sederhana dan analogi mengenai seluruh teknologi (Core Stack) yang kami gunakan:

⚙️ Backend Engine (Mesin Pemroses Data)

  • PHP Language (^8.2):
    Analogi: "Bahasa Percakapan Resmi Server".
    PHP adalah bahasa pemrograman dasar yang digunakan untuk menulis logika di balik layar. Bahasa ini bertugas menerima pendaftaran siswa, menghitung kuota bimbingan, dan memproses obrolan chat.
  • Laravel Framework (^11.9 || ^12.0):
    Analogi: "Peralatan Dapur Modern Siap Pakai".
    Laravel adalah kerangka kerja yang mempermudah penulisan PHP. Alih-alih membuat pengunci keamanan, pengirim surat, dan struktur data dari awal, Laravel menyediakan alat siap pakai agar sistem selesai lebih cepat, rapi, dan bebas celah keamanan.
  • Laravel Tinker (^2.9 || ^3.0):
    Analogi: "Remot Uji Coba Langsung".
    Alat ini memungkinkan pembuat program berkomunikasi langsung dengan database menggunakan perintah teks cepat untuk mengecek data siswa atau mereset password secara langsung tanpa harus membuka halaman web.

🎨 Frontend Stack (Desain & Tampilan Visual)

  • Tailwind CSS (^4.0.0):
    Analogi: "Palet Warna & Cetakan Desain Instan".
    Tailwind CSS mengatur warna tombol, bentuk kotak, ukuran teks, dan tata letak agar tampilan SIMBEKA terasa modern, bersih, dan nyaman saat dipandang mata di HP maupun laptop.
  • Vite Bundler (^8.0.0):
    Analogi: "Petugas Packing Ekspres".
    Vite bertugas mengompres (mengecilkan) file desain, gambar, dan kode Javascript ke dalam paket kecil agar halaman web tidak lemot saat dibuka oleh siswa.
  • @tailwindcss/vite & Laravel Vite Plugin:
    Analogi: "Saluran Pipa Air Otomatis".
    Menghubungkan sistem desain dengan mesin backend. Setiap kali pemrogram merubah letak tombol atau warna, perubahannya langsung mengalir ke server secara instan tanpa perlu mematikan dan menghidupkan ulang web.

📄 Data & Documents (Pengolahan Berkas)

  • Barryvdh DomPDF (^3.1):
    Analogi: "Mesin Cetak Surat Digital".
    DomPDF secara otomatis merangkum catatan bimbingan atau surat peringatan di aplikasi, lalu mencetaknya ke dalam berkas PDF (.pdf) resmi yang siap diprint atau dibagikan ke orang tua siswa.
  • PhpSpreadsheet (^5.7):
    Analogi: "Jembatan Pengisi Tabel Excel".
    Library ini bertugas membaca berkas Excel data siswa baru dari staf tata usaha sekolah, lalu secara otomatis memasukkan ribuan nama tersebut ke database SIMBEKA dalam sekejap tanpa perlu mengetik manual satu per satu.

🛠️ Dev Tools & Testing (Penjamin Kualitas & Sistem Deteksi)

  • Concurrently (^9.0.1):
    Analogi: "Operator Serbabisa (Multitasking)".
    Menjalankan server website, pemroses antrean chat, dan mesin kompresi desain secara bersamaan dalam satu klik, memudahkan proses pengembangan lokal.
  • PHPUnit (^10.5 || ^11.0):
    Analogi: "Robot Inspektur Kualitas (Quality Control)".
    Robot otomatis yang menguji seluruh tombol (tombol klaim kelas, kirim laporan, dll.) berulang kali secara otomatis untuk menjamin tidak ada tombol yang rusak sebelum aplikasi dipakai resmi oleh sekolah.
  • Laravel Pint (^1.27):
    Analogi: "Editor Ejaan & Tata Bahasa".
    Laravel Pint secara otomatis menyunting tulisan kode program agar rapi, seragam, dan mudah dibaca oleh pengembang lain yang ingin melanjutkan pemeliharaan sistem di masa mendatang.
  • Laravel Pail (^1.2.5):
    Analogi: "Pencatat CCTV Masalah".
    Laravel Pail memunculkan log (catatan) error secara langsung di layar komputer saat ada kesalahan sistem, sehingga pembuat aplikasi bisa mengetahui letak kerusakannya secara tepat detik itu juga.

🔒 Analogi Fitur Keamanan Tambahan

• Enkripsi AES-256 (Crypt): Kode pemulihan (Recovery Code) disembunyikan dalam brankas baja bersandi rumit. Hanya server yang memegang kunci pembukanya, mencegah peretas membaca kode tersebut sekalipun database bocor.

• Hashing Bcrypt: Mesin penghancur kertas satu arah. Password dihancurkan menjadi abu secara permanen. Sistem hanya mencocokkan kemiripan abunya saja saat login, menjaga kerahasiaan password mutlak bagi pengguna.

• Middleware & RBAC: Kartu akses ruangan. Siswa tidak bisa menyusup ke ruang bimbingan Guru BK, dan Guru BK tidak bisa menyamar menjadi Kepala Sekolah atau Super Admin.

• Self-Destruct Chat: Surat rahasia terbakar otomatis. Begitu kasus konsultasi online dinyatakan selesai (resolved), riwayat obrolan pribadi langsung dimusnahkan secara permanen dari layar web untuk menjaga aib siswa.
